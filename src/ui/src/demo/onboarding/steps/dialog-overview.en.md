The dialog has two jobs, and both matter.

The **left side** is where you define the project contract: what the task is, what resources or references already exist, and how far the first autonomous round should go. The **right side** is the generated kickoff prompt that will actually seed the workspace.

- Use the left side to shape the task precisely.
- Use the right side to verify that the system interpreted your intention correctly.
- If the right side looks wrong, fix the inputs before you launch.
